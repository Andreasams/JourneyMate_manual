│  actual code lives in _flutterflow_export/lib/custom_code/.                                                          │

│ where the README and BUNDLE.md say different things. During implementation, both files are read: BUNDLE.md for the   │
│ page context and the shared README for the component details.                                                        │
│                                                                                                                      │
│ Rule for every page session: Before implementing a custom widget/action/function, read its shared/ MASTER_README AND │
│  its source in _flutterflow_export/lib/custom_code/.                                                                 │

│                                                                                                                      │
│ One page, conditional button (NOT two pages):                                                                        │
│ - On init: calls getUserPreference('user_language_code')                                                             │
│ - If language pref exists → returning user → show only "Continue" button                                             │
│ - If language pref is null/empty → first time user → show BOTH "Continue" (English) AND "Fortsæt på dansk" (Danish)  │
│ buttons                                                                                                              │
│ - The "Fortsæt på dansk" button is hidden with if (_model.userLanguageCode == null || _model.userLanguageCode == '') │
│  — simple conditional render                                                                                         │
│                                                                                                                      │
│ Router decision (simplified):                                                                                        │
│ - Always route to /welcome on app launch — no token check needed                                                     │
│ - The page itself handles the new vs. returning distinction via stored language preference                           │
│                                                                                                                      │
│ On-load actions (parallel, from FlutterFlow source):                                                                 │
│ Parallel batch 1:                                                                                                    │
│   - getUserPreference('user_currency_code') → updateCurrencyWithExchangeRate()                                       │
│   - checkLocationPermission('welcomepage') [fire-and-forget]                                                         │
│   - getUserPreference('user_language_code')                                                                          │
│     - If empty → saveUserPreference('user_language_code', 'en'), setAppLanguage('en')                                │
│   - Set _buttonsMayShow = true → reveal buttons                                                                      │
│                                                                                                                      │
│ Parallel batch 2 (after batch 1):                                                                                    │
│   - getTranslationsWithUpdate(languageCode)                                                                          │
│   - getFiltersWithUpdate(languageCode) [fire-and-forget]                                                             │
│                                                                                                                      │
│ "Continue" button action:                                                                                            │
│ 1. Get current location                                                                                              │
│ 2. Call SearchCall (cityId, userLocation, '', languageCode)                                                          │
│ 3. markUserEngaged() [fire-and-forget]                                                                               │
│ 4. generateAndStoreFilterSessionId() [fire-and-forget]                                                               │
│ 5. Store searchResults + searchResultsCount in searchProvider                                                        │
│ 6. Track analytics 'page_viewed' [fire-and-forget]                                                                   │
│ 7. Navigate to /search                                                                                               │
│                                                                                                                      │
│ "Fortsæt på dansk" button action (first-time users only):                                                            │
│ Parallel:                                                                                                            │
│   - markUserEngaged, track analytics, saveUserPreference('user_currency_code', 'DKK'),                               │
│     saveUserPreference('user_language_code', 'da'), generateAndStoreFilterSessionId,                                 │
│     setAppLanguage('da')                                                                                             │
│   - getTranslationsWithUpdate('da'), getFiltersWithUpdate('da')                                                      │
│   - If locationStatus: SearchCall(da) → store results                                                                │
│   - If no location: requestLocationPermissionAndTrack → SearchCall(da) → store results                               │
│ Navigate to /search after 100ms delay                                                                                │
│                                                                                                                      │
│ State involved: searchStateProvider, localizationProvider, translationsProvider, filterProvider, locationProvider,   │
│ accessibilityProvider                                                                                                │
│                                                                                                                      │
│ Step 1: Implement welcome_page.dart following the action sequences above and BUNDLE.md                               │
│ Step 2: Wire button visibility conditional (_buttonsMayShow gates entire button column)                              │
│ Step 3: Wire router to always go to /welcome on launch (remove token check)                                          │
│ Step 4: Run translation key pass — scan for hardcoded strings, append to NEW_TRANSLATION_KEYS.sql                    │
│ Step 5: Run code review against the three-document spec (design + BUNDLE.md + providers)                             │
│ Step 6: Run flutter analyze — must be 0 warnings                                                                     │
│ Step 7: Commit — feat: implement welcome page with new/returning user conditional                                    │
